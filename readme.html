Website screenshot automation with Playwright

This project captures full-page screenshots of all relevant pages in a website, based on a root URL or a sitemap.xml. It is intended as a practical visual archive after go-live, with screenshot sets tailored to Beaver Builder breakpoints.

Recommended project structure

Keep the Python script and the virtual environment together in the same project folder.

playwright-screenshots/
├── screenshot.py
├── .venv/
└── screenshots/   (created after a run)
Important: you only install the dependencies once. After that, you usually just activate the virtual environment and run the script.

Usage

Basic

python screenshot.py --url https://example.com/sitemap.xml --variant basic --generate-index

Extended

python screenshot.py --url https://example.com/sitemap.xml --variant extended --generate-index

Custom output folder

python screenshot.py --url https://example.com/sitemap.xml --variant basic --output my-output --generate-index

Retry only previously failed URLs for the same domain

python screenshot.py --url https://example.com/sitemap.xml --variant basic --only-failed

Output structure

The script now creates a separate folder per domain, so screenshots from different websites do not get mixed together.

screenshots/
└── example.com/
    ├── report.json
    ├── report.csv
    └── 2026-03-27/
        ├── home/
        │   ├── desktop.png
        │   ├── medium.png
        │   ├── tablet.png
        │   └── mobile.png
        ├── about/
        │   └── ...
        └── index.html

That means each domain gets its own folder, and inside that folder you will find:

  • report.json
  • report.csv
  • a dated run folder
  • an optional index.html in the dated run folder when --generate-index is used

Visible progress in Terminal

The script now prints progress while running, for example:

[1/11] home -> viewport 2/4: medium (1199x900)
...

At the end it also prints a short summary with page counts and viewport success or failure counts.

Troubleshooting

python: command not found

Your virtual environment is probably not active, or it was moved and broke. Activate it again, or recreate .venv if needed.

No module named playwright

You are likely outside the active virtual environment. Run:

source .venv/bin/activate

The project folder was moved

If you moved the whole project folder and activation no longer works properly, recreate the virtual environment in the new location:

rm -rf .venv
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install playwright aiohttp
python -m playwright install
